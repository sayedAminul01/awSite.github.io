Essential Linux Terminal Commands: A Comprehensive Guide

Master the Linux command line with this detailed reference of the most widely used terminal commands for developers and sysadmins.

By: Alex Chen Updated: April 10, 2023 Reading Time: 20 min

The Linux command line is a powerful tool that can dramatically improve your productivity and control over your system. Whether you're a developer, system administrator, or power user, mastering these essential commands will help you work more efficiently. This guide covers the most widely used commands with practical examples and explanations.

File and Directory Operations

Commands for navigating, creating, and managing files and directories.

ls - List Directory Contents

ls [options] [directory]

Description

The ls command lists files and directories in the current working directory or a specified directory. It's one of the most frequently used commands for exploring the file system.

Common Examples

List files with details (permissions, owner, size, etc.):
ls -l
List all files including hidden ones (starting with '.'):
ls -a
List files with details and human-readable sizes:
ls -lh
List files sorted by modification time (newest first):
ls -lt

Pro Tip

Create an alias for ls -la in your .bashrc file to always see all files with details: alias ll='ls -la'

cd - Change Directory

cd [directory]

Description

The cd command changes the current working directory to the specified directory. Without arguments, it returns to your home directory.

Common Examples

Change to home directory:
cd
Change to a specific directory:
cd /var/www/html
Move up one directory level:
cd ..
Change to the previous directory:
cd -

cp - Copy Files and Directories

cp [options] source destination

Description

The cp command copies files and directories from a source to a destination location. It can copy single files, multiple files, or entire directory structures.

Common Examples

Copy a file to another location:
cp file.txt /backup/
Copy a file with a new name:
cp file.txt file_backup.txt
Copy a directory recursively (including all contents):
cp -r directory/ new_directory/
Copy with preservation of attributes:
cp -p file.txt backup/

Important

Be careful when copying files - the cp command will overwrite existing files without warning unless you use the -i (interactive) option.

File Viewing and Editing

Commands for viewing, searching, and editing file contents.

cat - Concatenate and Display Files

cat [options] [file(s)]

Description

The cat command reads files sequentially and writes them to standard output. It's commonly used to view file contents, create files, and concatenate multiple files.

Common Examples

Display file contents:
cat filename.txt
Display multiple files:
cat file1.txt file2.txt
Create a new file (type content, then Ctrl+D):
cat > newfile.txt
Number all output lines:
cat -n filename.txt

Pro Tip

For viewing large files, use less or more instead of cat as they provide paginated output that's easier to read.

grep - Search Text

grep [options] pattern [file(s)]

Description

The grep command searches for patterns in files or input. It's one of the most powerful and versatile commands for text processing and searching.

Common Examples

Search for a pattern in a file:
grep "error" logfile.txt
Case-insensitive search:
grep -i "warning" logfile.txt
Recursive search in directory:
grep -r "function_name" src/
Show line numbers with matches:
grep -n "TODO" *.py
Invert match (show non-matching lines):
grep -v "success" results.txt

find - Search for Files

find [path] [expression]

Description

The find command searches for files in a directory hierarchy based on various criteria like name, size, modification time, and permissions.

Common Examples

Find files by name:
find /home -name "*.txt"
Find files modified in the last 7 days:
find /var/log -mtime -7
Find and delete files:
find /tmp -name "*.tmp" -delete
Find files larger than 100MB:
find / -size +100M
Find files and execute a command on them:
find . -name "*.js" -exec chmod 644 {} \;

System Information and Monitoring

Commands for checking system status, resources, and performance.

ps - Process Status

ps [options]

Description

The ps command displays information about active processes. It's essential for monitoring what's running on your system and troubleshooting performance issues.

Common Examples

Show all processes for current user:
ps aux
Show processes in a tree format:
ps -ef --forest
Show processes for a specific user:
ps -u username
Show process by PID:
ps -p 1234

Pro Tip

For interactive process monitoring, use top or htop instead of ps as they provide real-time updating views of system processes.

top - System Monitoring

top [options]

Description

The top command provides a dynamic real-time view of running system processes. It displays system summary information and a list of processes ordered by CPU usage.

Common Examples

Start top with custom update interval (3 seconds):
top -d 3
Show processes for a specific user:
top -u username
Start top in batch mode (for scripting):
top -b -n 1

Interactive Commands in top

While running top, you can use these keys: P (sort by CPU), M (sort by memory), k (kill process), q (quit).

df - Disk Space Usage

df [options] [file]

Description

The df command reports file system disk space usage. It shows the amount of disk space available and used on mounted file systems.

Common Examples

Show disk usage in human-readable format:
df -h
Show disk usage for a specific filesystem:
df -h /home
Show inode usage instead of block usage:
df -i
Show all filesystems including dummy ones:
df -a

Networking

Commands for network configuration, troubleshooting, and monitoring.

ping - Test Network Connectivity

ping [options] destination

Description

The ping command sends ICMP ECHO_REQUEST packets to network hosts to test connectivity and measure round-trip time.

Common Examples

Ping a host with a count of 5 packets:
ping -c 5 google.com
Ping with specific interval (2 seconds):
ping -i 2 example.com
Ping with timestamp for each packet:
ping -D 8.8.8.8
Flood ping (for stress testing):
ping -f localhost

Important

Some networks block ICMP packets for security reasons, so a failed ping doesn't always mean the host is down.

netstat - Network Statistics

netstat [options]

Description

The netstat command displays network connections, routing tables, interface statistics, masquerade connections, and multicast memberships.

Common Examples

Show all listening ports:
netstat -tulpn
Show routing table:
netstat -r
Show network interface statistics:
netstat -i
Show all established connections:
netstat -t

Pro Tip

On modern systems, ss is often preferred over netstat as it's faster and provides more information.

ssh - Secure Shell

ssh [options] [user@]hostname [command]

Description

The ssh command provides a secure encrypted connection between two hosts over an insecure network. It's the standard for remote administration of Linux systems.

Common Examples

Connect to a remote server:
ssh username@server.example.com
Connect with a specific port:
ssh -p 2222 user@hostname
Execute a command on remote host:
ssh user@hostname 'ls -la'
Tunnel a local port to remote host:
ssh -L 8080:localhost:80 user@hostname

Mastering the Linux Command Line

These essential Linux commands form the foundation of effective system administration and development work. With practice, you'll find yourself combining these commands in powerful ways using pipes (|), redirection (>, >>), and command substitution ($()).

Remember that the best way to learn is through practice. Set up a test environment and experiment with these commands to build your confidence and understanding of the Linux command line.
